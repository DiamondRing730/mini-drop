# 智能归因评测报告

## 评测目标

验证归因引擎能否找出自耗时最高的函数、提供建议、留下工具调用证据，并让独立校验器
复核结论中的函数名和自耗时比例。评测集包含递归计算、数值循环、读取路径和解析路径
四类确定性 profile。

## 复现命令

```bash
python3 scripts/evaluate_attribution.py --backend offline
```

DeepSeek 会产生外部 API 请求和费用，因此不会在普通测试中自动执行。需要评测时显式运行：

```bash
set -a
source .env
set +a
python3 scripts/evaluate_attribution.py --backend deepseek
```

## 离线归因结果

| 指标 | 结果 |
|---|---:|
| 用例数 | 4 |
| Top-1 根因函数准确率 | 100% |
| 结论数值校验通过率 | 100% |
| 优化建议覆盖率 | 100% |
| Profile 工具轨迹覆盖率 | 100% |

离线引擎是确定性规则基线，因此结果可重复。DeepSeek 结果应在配置密钥后单独运行并保存
命令输出；在未实际调用前不填写或虚构其准确率。

## 局限

- 当前评测集是合成 profile，适合验证数据契约和防幻觉机制，不等价于真实生产根因集。
- 校验器验证函数存在性与自耗时比例，不判断优化建议在业务语境中是否一定有效。
- 后续应增加真实故障样本、人工标注根因和多次 DeepSeek 运行的稳定性统计。
