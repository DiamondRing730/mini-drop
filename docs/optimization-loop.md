# 可验证性能优化闭环

## 目标

把一次性的“采样 → 归因”扩展为：

```text
优化前采样 → 修改代码/配置 → 优化后采样 → profile 差分 → 独立校验
```

在任意已完成的 perf/py-spy 任务详情页中，当前任务自动作为“优化后”，用户选择一个
同采集器、已分析完成的历史任务作为“优化前”，点击“生成可验证效果报告”。

## 输出

- 优化前后首要热点占比及变化（百分点）。
- 按绝对变化排序的函数差分柱状图，绿色为占比下降、红色为占比上升。
- 以优化后调用栈为布局的差分火焰图；每个 frame 的 tooltip 同时显示优化前、优化后与差值。
- 独立校验报告：绕过报告生成逻辑，从两个原始 `tree.json` 重新计算每个函数的占比和差值。
- 持久化的 `comparison-<baseline>.json` 与 `diff-flamegraph-<baseline>.svg` 任务产物。

## 比较规则

- 只比较同一种采集器的两个 `DONE + analysis DONE` 任务。
- 使用自耗时占总采样数的百分比，消除两次样本数不同带来的直接计数偏差。
- 自动把 `fib (workload.py:10)`、`fib (workload.py:11)` 等行级 frame 合并为函数 `fib`。
- 变化达到 ±1 个百分点时标记为改善/恶化，否则标记为稳定。
- 两边最小样本数 ≥500 / ≥100 / <100 时，置信度分别为 high / medium / low。

## 结果边界

profile 占比差分能证明热点分布发生变化，但不能单独证明整体延迟或吞吐提升。正式优化实验应保持
负载、采集器、采样参数一致，并结合业务吞吐或延迟指标判断整体效果。系统在报告中固定展示这项限制，
避免把“热点占比下降”夸大为“整体性能一定提升”。
